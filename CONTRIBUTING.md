# Contribuer · Contributing

**[Français](#français) · [English](#english)**

## Français

### Signaler une erreur

Si tu trouves une erreur technique, une imprécision ou un lien cassé, ouvre une [Issue](https://github.com/DialloDBA/how-it-works/issues) avec le label `correction`. Précise le fichier concerné et ce qui est incorrect.

### Proposer un sujet

Si tu veux voir un sujet documenté, ouvre une [Issue](https://github.com/DialloDBA/how-it-works/issues) avec le label `new-article`. Les meilleures propositions sont les sujets peu documentés en français.

### Si tu veux écrire un article

Quelques règles à respecter :

- Noms de fichiers en kebab-case sans accents : `idempotency-keys.md`, pas `Idempotency_Keys.md`
- Chaque article doit exister en FR et EN, avec un lien croisé dans le frontmatter
- Mettre `status: draft` pendant l'écriture, passer à `status: published` quand c'est terminé
- Un article = un sujet précis. Pas de survol général.

Pour soumettre : fork → branche → article → pull request.

## English

### Report an error

If you find a technical mistake, an inaccuracy or a broken link, open an [Issue](https://github.com/DialloDBA/how-it-works/issues) with the `correction` label. Include the file path and what is wrong.

### Suggest a topic

If you want to see a topic covered, open an [Issue](https://github.com/DialloDBA/how-it-works/issues) with the `new-article` label. The most useful suggestions are topics that are poorly documented in English or French.

### If you want to write an article

A few rules:

- File names in kebab-case, no accented characters: `idempotency-keys.md`, not `Idempotency_Keys.md`
- Every article must exist in both FR and EN, with a cross-link in the frontmatter
- Use `status: draft` while writing, switch to `status: published` when done
- One article = one precise topic. No broad overviews.

To submit: fork → branch → article → pull request.
